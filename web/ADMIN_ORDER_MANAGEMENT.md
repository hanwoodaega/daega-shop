# 관리자 주문 관리 시스템

## 개요

관리자가 주문을 확인하고 상태를 변경할 수 있는 시스템입니다.

## 배송 유형별 주문 상태

### 1. 📦 택배배달 (regular)
- **결제 완료** (paid) → 자동으로 택배사에서 처리
- **배송 중** (shipped) → 택배사 자동 업데이트
- **배송 완료** (delivered) → 택배사 자동 업데이트
- ⚠️ **관리자가 상태를 변경할 수 없음** (택배사 자동 처리)

### 2. 🚗 퀵배달 (quick)
- **결제 완료** (paid) → 관리자가 **배송 중** 또는 **취소**로 변경 가능
- **배송 중** (shipped) → 관리자가 **배송 완료**로 변경 가능
- **배송 완료** (delivered) → 완료 (변경 불가)

### 3. 🏪 매장 픽업 (pickup)
- **결제 완료** (paid) → 관리자가 **준비 중** 또는 **취소**로 변경 가능
- **준비 중** (shipped) → 관리자가 **완료**로 변경 가능
- **완료** (delivered) → 완료 (변경 불가)

## 주문 취소 규칙

### 고객
- **결제 완료 상태에서만** 주문 취소 가능
- 준비 중, 배송 중, 완료 상태에서는 취소 불가능
- 취소 시 자동으로 환불 처리 시작

### 관리자
- 결제 완료 상태에서 취소로 변경 가능
- 이미 준비/배송 중인 주문은 취소 불가

## 환경 변수 설정

### `.env.local` 파일에 추가

```bash
# Supabase Service Role Key (관리자 권한으로 데이터베이스 접근)
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key_here

# 관리자 비밀번호 (기존 관리자 로그인과 동일)
ADMIN_PASSWORD=admin123
```

### Supabase Service Role Key 찾기

1. Supabase 프로젝트 대시보드 접속
2. **Settings** → **API** 메뉴
3. **Project API keys** 섹션
4. **`service_role` key** 복사 (⚠️ 절대 공개하지 말 것!)

## 관리자 페이지 접근

### 1. 관리자 로그인
```
URL: http://localhost:3000/admin/login
ID: admin
PW: admin123
```

### 2. 주문 관리 페이지 접근
```
URL: http://localhost:3000/admin/orders
```

또는 관리자 페이지에서 **"주문 관리"** 버튼 클릭

## 기능

### 📅 날짜별 조회
- 특정 날짜의 주문만 필터링
- 기본값: 오늘 날짜

### 🚚 배송 유형별 필터
- 전체 / 매장 픽업 / 퀵배달 / 택배배달

### 📊 주문 상태별 필터
- 전체 / 결제 대기 / 결제 완료 / 배송 중(준비 중) / 배송 완료(완료) / 주문 취소

### 📈 통계 요약
- 전체 주문 수
- 배송 유형별 주문 수 (픽업 / 퀵배달 / 택배배달)

### 🔄 상태 변경
- 퀵배달, 픽업 주문의 상태를 관리자가 직접 변경
- 택배배달은 자동 처리되므로 변경 불가

## 주문 정보 표시

각 주문에는 다음 정보가 표시됩니다:

### 고객 정보
- 이름
- 이메일
- 전화번호

### 주문 상품
- 상품명
- 수량
- 가격

### 배송 정보
- 배송 유형
- 수령인
- 연락처
- 배송 주소
- 배송 요청사항
- 픽업/배달 시간 (해당 시)

### 결제 정보
- 총 결제 금액

## 사용 예시

### 매장 픽업 주문 처리 순서

1. 고객이 매장 픽업으로 주문 (상태: **결제 완료**)
2. 관리자가 주문 확인 후 상품 준비 시작 → **준비 중**으로 변경
3. 상품 준비 완료, 고객에게 픽업 가능 알림 → **완료**로 변경

### 퀵배달 주문 처리 순서

1. 고객이 퀵배달로 주문 (상태: **결제 완료**)
2. 관리자가 배달 기사에게 배달 요청 → **배송 중**으로 변경
3. 배달 완료 후 → **배송 완료**로 변경

## 보안 주의사항

⚠️ **SUPABASE_SERVICE_ROLE_KEY는 절대 공개하지 마세요!**

- 이 키는 Supabase의 모든 보안 정책을 우회할 수 있습니다
- `.env.local` 파일은 git에 커밋하지 마세요
- 프로덕션 환경에서는 환경 변수로 안전하게 관리하세요

## 문제 해결

### "권한이 없습니다" 오류
→ `.env.local`에 `ADMIN_PASSWORD`가 올바르게 설정되어 있는지 확인
→ 관리자 로그인 페이지에서 다시 로그인

### "주문 조회에 실패했습니다" 오류
→ `.env.local`에 `SUPABASE_SERVICE_ROLE_KEY`가 올바르게 설정되어 있는지 확인
→ 개발 서버를 재시작 (`npm run dev`)

### 상태 변경이 안 됨
→ 택배배달은 상태 변경이 불가능합니다 (자동 처리)
→ 픽업/퀵배달만 관리자가 상태 변경 가능

## API 엔드포인트

### GET `/api/admin/orders`
주문 목록 조회

**Query Parameters:**
- `delivery_type`: pickup | quick | regular
- `date`: YYYY-MM-DD
- `status`: pending | paid | shipped | delivered | cancelled

**인증:** 쿠키 기반 (`admin_auth` 쿠키 필요)

### PATCH `/api/admin/orders`
주문 상태 변경

**Body:**
```json
{
  "orderId": "uuid",
  "status": "shipped" | "delivered" | "cancelled"
}
```

**인증:** 쿠키 기반 (`admin_auth` 쿠키 필요)

## 고객 주문내역 페이지

고객이 보는 주문내역 페이지도 배송 유형에 따라 다르게 표시됩니다:

### 픽업 주문
- 결제 완료 → 준비 중 → 완료
- 결제 완료 상태에서만 취소 가능

### 퀵배달/택배배달
- 결제 완료 → 배송 중 → 배송 완료
- 결제 완료 상태에서만 취소 가능
- 배송 중/배송 완료 상태에서 "배송조회" 버튼 표시
