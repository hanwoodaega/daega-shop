# RLS 활성화 분석

## ✅ RLS 활성화 필요 (사용자별 데이터)

1. **order_items** - 사용자는 자신의 주문 항목만 조회/생성
2. **orders** - 사용자는 자신의 주문만 조회/생성/수정
3. **point_history** - 사용자는 자신의 포인트 내역만 조회
4. **user_coupons** - 사용자는 자신의 쿠폰만 조회/사용
5. **user_terms** - 사용자는 자신의 약관 동의 내역만 조회
6. **user_points** - 사용자는 자신의 포인트 정보만 조회

## ⚠️ RLS 활성화 시 주의 필요 (공개 데이터)

### products 테이블
- **현재 사용**: 모든 사용자가 상품 조회 가능 (로그인 불필요)
- **RLS 활성화 조건**: 
  - 정책이 "모든 사용자가 활성 상품 조회 가능"이어야 함
  - `status = 'deleted'`인 상품은 제외되어야 함
- **주의**: 정책이 제대로 설정되지 않으면 상품 조회가 실패할 수 있음

### coupons 테이블
- **현재 사용**: 모든 사용자가 활성 쿠폰 조회 가능
- **RLS 활성화 조건**:
  - 정책이 "모든 사용자가 활성 쿠폰 조회 가능"이어야 함
  - 관리자는 모든 쿠폰 조회/생성/수정 가능해야 함
- **주의**: 정책이 제대로 설정되지 않으면 쿠폰 조회가 실패할 수 있음

## 권장 사항

1. **먼저 RLS 정책 확인**: Supabase 대시보드에서 각 테이블의 RLS 정책을 확인
2. **products, coupons 정책 확인**:
   - `products`: "모든 사용자가 SELECT 가능" 정책이 있는지 확인
   - `coupons`: "모든 사용자가 활성 쿠폰 SELECT 가능" 정책이 있는지 확인
3. **테스트**: RLS 활성화 후 애플리케이션 동작 테스트
